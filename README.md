Описание

Данный проект представляет собой высокопроизводительную неблокирующую (lock-free) хеш-таблицу на языке C, 
использующую атомарные операции Compare-And-Swap (CAS) для работы в многопоточных средах. 
В отличие от традиционных хеш-таблиц, использующих мьютексы или критические секции, 
данная реализация позволяет потокам работать параллельно без блокировок и взаимных ожиданий, 
что значительно повышает масштабируемость и производительность при высокой конкуренции.

Проект также включает блокирующую версию хеш-таблицы на критических секциях для сравнения производительности и 
наглядной демонстрации преимуществ lock-free подхода в реальных сценариях использования.

Структуры данных:
- LockFreeNode — узел связного списка, хранящий ключ, значение и указатель на следующий узел. 
Поле key становится NULL после удаления узла, что служит маркером удаления (tombstone).
- LockFreeHashTable — основная структура неблокирующей хеш-таблицы, содержащая массив корзин, размер, маску для быстрого индекса, 
указатель на хеш-функцию и атомарный счётчик элементов.
- BlockingHashTable — структура блокирующей хеш-таблицы для сравнения, использующая критическую секцию для синхронизации.

Основные функции:
- Создание таблицы: выделяет память, инициализирует структуру, создаёт массив корзин размером, приведённым к степени двойки.
- Вставка элемента: использует CAS-цикл для атомарной вставки. Если ключ уже существует, обновляет значение.
При неудачной CAS-операции освобождает созданный узел и повторяет попытку.
- Поиск элемента: выполняет безопасное чтение с локальным копированием указателя на ключ для защиты от удаления узла другим потоком.
- Удаление элемента: использует CAS для атомарного удаления узла из цепочки, помечая удалённый узел ключом NULL.
- Безопасное освобождение: корректно освобождает память всех узлов при уничтожении таблицы.

Сравнение подходов к синхронизации
- Однопоточная версия, которая была реализована в предыдущей работе, не использует синхронизацию, 
обеспечивает максимальную производительность, но небезопасна в многопоточных средах. Подходит только для однопоточных приложений,
где нет конкуренции за данные.
- Блокирующая версия использует критические секции или мьютексы для синхронизации.
Поток блокируется при доступе к таблице, что может приводить к взаимным блокировкам (deadlocks) и инверсии приоритетов.
Отличается простотой реализации и хорошо работает при низкой конкуренции, но плохо масштабируется с ростом числа потоков.
- Неблокирующая версия использует CAS-операции для атомарного обновления данных.
Потоки не блокируются, обеспечивая прогресс системы даже при высокой конкуренции.
Исключает взаимные блокировки, показывает лучшую масштабируемость, но требует более сложной реализации и управления памятью.

Теоретическая часть

Хеш-функция преобразует входной ключ в индекс массива. 
В данной реализации используется хеш-функция FNV-1a (Fowler-Noll-Vo вариант 1a), 
выбранная за отличные распределительные свойства и высокую производительность. 
Алгоритм FNV-1a прост и эффективен: для каждого байта данных выполняется операция XOR с текущим хешем, 
затем умножение на простое число FNV_prime.

Корзины — это массив, где каждый элемент является началом связного списка. 
Размер таблицы всегда является степенью двойки для оптимизации вычисления индекса через побитовую операцию AND вместо медленного деления.

Цепочки — связные списки для разрешения коллизий. 
Когда два разных ключа дают одинаковый хеш-индекс, они сохраняются в одной корзине в виде связного списка, 
что обеспечивает простую и надёжную обработку коллизий.

Неблокирующая (lock-free) хеш-таблица — это структура данных, 
которая позволяет безопасно работать с ней в многопоточной среде без использования традиционных механизмов синхронизации (мьютексов, семафоров). 
Вместо этого корректность при конкурентном доступе обеспечивают атомарные операции и специальные алгоритмы.

Lock-free хеш-таблицы особенно полезны в высоконагруженных системах, где критически важна пропускная способность и параллелизм, 
например, в сетевых серверах, базах данных или игровых приложениях.

Ключевой инструмент — атомарные операции, которые позволяют выполнить условное изменение данных как неделимую единицу. 
Один из самых распространённых примитивов — Compare-And-Swap (CAS). 
Идея в том, что перед модификацией система проверяет: совпадает ли текущее значение с ожидаемым. 
Только если да — замена выполняется, иначе поток «понимает», что его попытка провалилась, и повторяет попытку позже.
В неблокирующей хеш-таблице CAS используется для:
- Атомарной вставки нового узла в начало цепочки
- Атомарного удаления узла из цепочки
- Безопасного обновления указателей без блокировок


У неблокирующих хеш-таблиц есть как преимущества, так и серьёзные сложности.

Преимущества:

- Высокая производительность в системах с высокой степенью параллелизма, так как потоки выполняются независимо, не ожидая освобождения блокировок.

- Устойчивость к отказам потоков: даже если один поток прерывается, другие могут продолжить выполнение.

Сложности: 
- Проблемы с удалением элементов. Физически удалить элемент из структуры данных в lock-free среде сложно.
Часто используется двухфазное удаление: сначала элемент атомарно помечается как «удаленный» (логическое удаление),
а позже, когда гарантируется, что на него никто не ссылается, происходит физическая очистка памяти. 
- Сложности с масштабированием.
Операции изменения размера таблицы (ресайз) в неблокирующих алгоритмах часто требуют сложной синхронизации и могут снижать производительность.
- Риск высокой конкуренции.
При очень высокой нагрузке на таблицу потоки могут часто конфликтовать при попытках вставки или обновления,
что приводит к множественным повторным попыткам и снижению пропускной способности.

Программа содержит два основных теста, которые проверяют корректность работы неблокирующей хеш-таблицы и сравнивают её производительность с блокирующей версией.

В программе реализованы следующие тестовые функции:

test_lockfree_hash_table()

Тест базовых операций неблокирующей хеш-таблицы.
Эта функция проверяет корректность работы основных операций в однопоточном режиме:
- Вставка трёх тестовых элементов
- Поиск элемента по ключу
- Удаление элемента
- Вывод содержимого таблицы для визуального контроля

test_performance_comparison_fast()

Тест сравнения производительности

Эта функция выполняет многопоточное сравнение двух версий хеш-таблицы:

- Тестирование блокирующей версии с критическими секциями
- Тестирование неблокирующей версии с CAS-операциями
- Замер времени выполнения 8 потоков по 200000 операций каждый
- Вычисление ускорения неблокирующей версии
